# Тестирование синхронизации Obstacle Race

## Чек-лист для тестирования:

### 1. ✅ Игра с ботом (локальный режим)
- [ ] Счёт обновляется правильно после каждого раунда
- [ ] Нет "прыжков" счёта
- [ ] Анимации проигрываются полностью
- [ ] Способности работают корректно
- [ ] Овертайм запускается правильно

### 2. ✅ PvP игра (нормальный интернет)
- [ ] Счёт одинаковый у обоих игроков
- [ ] Раунды начинаются синхронно
- [ ] Результаты раундов показываются одновременно
- [ ] Таймер синхронизирован
- [ ] Способности соперника видны корректно

### 3. ✅ PvP игра (плохой интернет)
- [ ] Fallback HTTP подхватывает если WebSocket отвалился
- [ ] Нет зависаний на экране "Соперник думает..."
- [ ] Игра продолжается даже при задержках
- [ ] Счёт восстанавливается после переподключения

### 4. ✅ Критические моменты
- [ ] Расстановка ловушек - оба игрока переходят к игре
- [ ] Первый раунд - счёт 0:0
- [ ] Середина игры - счёт корректный
- [ ] Овертайм - счёт сохраняется, новые ловушки
- [ ] Конец игры - правильный победитель

### 5. ✅ Способности
- [ ] Рентген - показывает ловушку/безопасно
- [ ] Удвоение - +2 при успехе, -1 при провале
- [ ] Саботаж - отменяет очко соперника
- [ ] Способности блокируются после использования
- [ ] В овертайме способности отключены

## Как тестировать:

### Тест 1: Быстрая игра с ботом
```
1. Открой игру
2. Нажми "Играть с ботом"
3. Расставь ловушки
4. Играй 7 раундов
5. Проверь что счёт обновляется плавно
```

### Тест 2: PvP на двух устройствах
```
1. Открой игру на двух телефонах/вкладках
2. Создай комнату на обоих
3. Играй одновременно
4. Сравнивай счёт на обоих экранах после каждого раунда
5. Счёт должен быть ИДЕНТИЧНЫМ
```

### Тест 3: Проверка логов
```
1. Открой DevTools (F12)
2. Вкладка Console
3. Играй игру
4. Смотри логи:
   ✅ "Processing new state" - новое состояние
   ⏭️ "Skipping duplicate state" - дубль пропущен
   📡 "WebSocket update received" - WebSocket работает
```

### Тест 4: Стресс-тест
```
1. Играй очень быстро (делай ходы за 1-2 секунды)
2. Используй способности каждый раунд
3. Доведи до овертайма
4. Проверь что нет лагов и рассинхрона
```

## Ожидаемые логи в консоли:

### Нормальная игра:
```
✅ Supabase Realtime initialized for Obstacle Race
🔌 Starting Realtime WebSocket for obstacle race room: xxx
WebSocket status: SUBSCRIBED
✅ WebSocket connected! Fetching current state...
✅ Processing new state: active:placing_traps:0:0:0:0 phase: placing_traps status: active
✅ Processing new state: active:running:0:0:0:0 phase: running status: active
📡 WebSocket update received: {...}
✅ Processing new state: active:running:0:0:1:0 phase: running status: active
⏭️ Skipping duplicate state: active:running:0:0:1:0
✅ Processing new state: active:running:1:0:1:0 phase: running status: active
```

### Если есть дубли (это нормально):
```
📡 WebSocket update received: {...}
⏭️ Skipping duplicate state: active:running:2:0:2:0
```

### Если WebSocket отвалился (fallback работает):
```
WebSocket status: CLOSED
(через 3 секунды)
✅ Processing new state: active:running:3:0:3:0 phase: running status: active
```

## Признаки проблем:

### ❌ Рассинхрон счёта:
- Счёт разный на двух устройствах
- Счёт "прыгает" (3→2→3)
- Счёт не обновляется

### ❌ Зависание:
- Застрял на "Соперник думает..."
- Таймер дошёл до 0 но ничего не происходит
- Кнопки не реагируют

### ❌ Дублирование:
- Анимация проигрывается дважды
- Очки начисляются дважды
- Раунд повторяется

## Если нашёл баг:

1. Открой DevTools Console
2. Скопируй все логи
3. Сделай скриншот обоих экранов (если PvP)
4. Опиши что делал перед багом
5. Отправь всё разработчику

## Автоматическая защита:

Код теперь имеет защиту на КАЖДОМ уровне:

1. **Дедупликация по хешу** - пропускает одинаковые состояния
2. **Debounce WebSocket** - блокирует шторм обновлений на 100мс
3. **Флаг roundAnimating** - блокирует обновления во время анимации
4. **Маркеры раундов** - не обрабатывает старые результаты
5. **Атомарное обновление счёта** - счёт обновляется один раз в начале
6. **Проверка перед стартом раунда** - не запускает если идёт анимация

Всё это работает АВТОМАТИЧЕСКИ, без участия игрока!
